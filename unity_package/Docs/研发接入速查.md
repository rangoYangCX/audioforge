# AudioForge Unity SDK 研发接入速查

当前文档同步日期：2026-05-14

这份文档给拿到包的 Unity / 客户端研发同学快速确认三件事：

- 这包怎么最快接进项目。
- 当前契约和一期相比最重要的变化是什么。
- 出问题优先查哪些点。

## 1. 最短接入路径

1. 把整个 `AudioForgeUnityPackage-<version>/` 作为本地 UPM 包接入 Unity。
2. 把工具导出的 `AudioData.json`、`AudioManifest.json` 和 `Assets/` 音频目录放到 `Assets/StreamingAssets/AudioForge/`。
3. 初始化 `AudioForgeRuntime`，触发一个真实 Event Id。
4. 用 `AudioForgeRuntimeDebugPanel` 或日志确认事件确实被命中。

## 2. 当前最重要的契约变化

- 当前正式契约是 `SchemaVersion = 3`。
- 运行时主结构已经固定为 `AudioObjects + Events[AudioId]`。
- Event 固定为动作层，Audio Object 固定为声音层。
- 当前包内 runtime 已支持 GameParameters、StateGroups、SwitchGroups、Audio / Bus 级 GameSync 绑定，以及 emitter context。

## 3. 接入时别再踩的坑

1. 不要读取 `.afproj`。
2. 不要按旧结构读取事件内嵌 `Audio`。
3. 不要只复制 `AudioData.json`，漏掉 `Assets/` 运行时音频目录。
4. 不要在业务层重写一套随机、序列、连击规则。
5. 不要忽略 Event 和 Audio Object 分层后的语义变化。

## 4. 遇到问题先查

- `AudioData.json` 是否实际加载成功。
- Event Id 是否存在。
- Event 的 `AudioId` 是否能定位到目标 Audio Object。
- 资源文件路径是否正确。
- 当前是否被冷却、实例数或 Bus 状态拦住。
