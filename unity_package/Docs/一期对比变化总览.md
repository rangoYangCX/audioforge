# AudioForge Unity SDK 包内一期对比变化总览

当前文档同步日期：2026-05-12

这份文档给拿到 SDK 包的 Unity 同学快速看：相对第一期，这个包现在新增了什么，哪些只属于编辑器，哪些需要你在 Unity 侧至少理解到位。

## 先看结论

- 当前最值得关注的运行时语义新增点只有一个：`PlayMode` 现在明确包含 `OneShot`。
- 包里附带的运行时代码目录结构没有被拆成第二套；主要新增的是文档解释、一期对比入口和最新验证材料。
- 对象浏览器三分页、绑定弹窗、`Enabled / Active` 编辑态和智能总线分配设置都不要求 Unity 侧新增字段消费。

## 一期 vs 当前

| 项目 | 第一期 | 当前 | Unity 侧动作 |
| --- | --- | --- | --- |
| `PlayMode` | 三类：`Random / Sequence / Combo` | 四类：新增 `OneShot` | 确认逻辑没有写死三类 |
| 导出 `Clips` | 直接消费导出结果 | 仍直接消费导出结果，但已是工具过滤后的有效集合 | 继续按导出结果消费 |
| 绑定编辑方式 | 一期没有当前这套收口 | 事件树展开后通过 bindings 弹窗编辑 | 无需改 |
| 对象浏览方式 | 旧工程树心智 | 总线树 / 源音频树 / 事件树三分页 | 无需改 |
| 工程默认总线策略 | 固定默认值为主 | 新增按命名智能分配总线开关 | 无需改 |
| 自动建事件 | 无当前 OneShot 默认策略 | 单源导入自动建 `OneShot` | 只需读懂导出的 `PlayMode` |

## 这包里建议怎么读

1. 先看这份差异总览，确认你需要关注的是语义边界，而不是去猜 SDK 目录有没有被重写。
2. 再看 `QuickStart.md`，用最短路径跑通第一声。
3. 然后看 `Canonical/UnitySDK对接规范.md`，确认 `OneShot`、有效 Clip 集合和 editor-only 边界理解一致。
4. 最后看 `Verification/` 里的报告和签收摘要，确认这包是按什么基线验证过的。
