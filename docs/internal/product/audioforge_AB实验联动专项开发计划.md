# AudioForge AB 实验联动专项开发计划

更新时间：2026-05-17

## 1. 专项目标

本专项以 AB 实验为核心场景，目标不是继续补单点功能，而是把实验工作流真正打通到事件编辑、资源管理、试听、校验、导出与恢复。

专项成功标准：

1. 用户能在任意关键页面明确知道当前正在查看底板还是方案。
2. 用户能快速判断当前对象属于底板继承、实验修改还是实验新增。
3. 用户能沿着固定流程完成：选方案、编辑、试听、校验、导出、回滚。
4. 用户不需要记住系统结构，也不需要在多个页面之间来回找入口。

## 2. 当前基线

当前版本已经具备以下基础能力：

1. Experiment 集成协调层已经拆出到 experiment_integration_controller。
2. 主控制器已经可以构建事件、Audio、Source 的归属标签。
3. 主窗口已经有顶部来源提示和自动保存设置。
4. 实验面板已经支持任务/方案树、差异预览和最近导出记录。
5. RecoveryService 已经支持自动保存历史快照。

因此本专项从“操作体验收敛”和“跨模块联动”出发，不再从底层模型重做。

## 3. 专项范围

### 3.1 包含

1. AB 实验全局上下文展示。
2. 事件 / Audio / Source 的实验归属统一表达。
3. 试听、导出、恢复、校验的实验态操作收敛。
4. 实验页内的最近快照与最近导出复查。
5. 关键路径的交互简化和主动作设计。

### 3.2 不包含

1. 方案对方案主流程。
2. 多人协作冲突解决。
3. Unity 端 AB 编排逻辑。
4. MainController 全量拆分重构。

## 4. 核心产品原则

1. 单一主动作：每个页面只突出当前最该做的一步。
2. 上下文前置：先说明正在编辑什么，再给动作入口。
3. 连续路径：编辑、试听、校验、导出、回滚形成闭环。
4. 结果可解释：每一步执行后都能回答“做了什么、影响了什么、下一步建议是什么”。

## 5. 研发拆分

### P0 实验上下文层

目标：让用户始终知道当前实验状态。

交付：

1. 全局实验状态工作条。
2. 对象级差异摘要。
3. 统一的底板 / 方案 / 归属文案与枚举。

### P1 实验编辑联动

目标：在编辑页面内直接解释实验改动。

交付：

1. 当前 Event 相对底板差异摘要。
2. Audio / Source 归属状态强化。
3. 对象级“查看差异 / 试听底板”入口。

### P2 实验交付联动

目标：让试听、校验、导出形成稳定路径。

交付：

1. 底板试听入口。
2. 导出前确认向导。
3. 导出历史与影响范围复查。

### P3 实验恢复联动

目标：降低试错成本。

交付：

1. 实验页最近快照列表。
2. 恢复最近快照快捷入口。
3. 自动保存记录与实验上下文绑定显示。

## 6. 本轮开发范围

本轮直接实现一批可闭环的 UX 功能，作为专项第一阶段可交付版本：

1. 实验状态工作条。
2. 对象级差异摘要。
3. 底板试听入口。
4. 实验页最近快照列表。
5. 导出前确认向导。

## 8. 验收标准

1. 进入实验方案后，主界面能明确显示底板、任务、方案和当前对象差异摘要。
2. 用户可直接从主界面试听底板版本，无需先切换回底板工程。
3. 实验页能查看当前方案相关的最近自动保存记录并触发恢复。
4. 导出前系统会展示当前方案差异摘要并请求确认。
5. 相关 GUI / controller / recovery 回归测试通过。

## 9. 风险与控制

1. 风险：MainController 继续堆厚。
控制：把实验动作编排仍放在 experiment_integration_controller，主控制器只负责上下文聚合和全局刷新。

2. 风险：UI 改动过多导致回归面变大。
控制：优先增量添加工作条和摘要，不重做整页结构。

3. 风险：试听底板引入上下文错乱。
控制：底板试听只构造临时试听会话，不切换当前工程和选择状态。

## 10. 建议后续路线

完成本轮后，下一阶段优先推进：

1. 对象级差异抽屉。
2. A/B 试听快切。
3. 结果中心实验过滤器。
4. 实验时间线视图。
