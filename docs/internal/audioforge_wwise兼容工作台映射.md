# AudioForge Wwise 兼容工作台映射

## 目的

这份文档定义 AudioForge 下一阶段在本地桌面工具层的界面演进方向：

- 目标不是像素级复制 Wwise。
- 目标是复用 Wwise 用户最熟悉的工作台心智与任务路径。
- 范围只限本地 AudioForge 工具，不改 Unity 运行时输入边界。

## 不可变边界

本轮优化必须保持以下边界不变：

- 不修改 AudioData.json、AudioManifest.json、AudioEventID.cs、BuildReport.json 的既有字段语义。
- 不修改运行时音频资源目录结构和 Unity 侧读取路径。
- 不要求 Unity 侧读取 .afproj 或依赖 Python 运行环境。
- 不把编辑器内部布局状态、筛选状态或临时交互状态写进导出结果。

换句话说，这一轮只改“怎么编辑”，不改“导出给 Unity 什么”。

## 设计原则

### 1. 认知兼容优先于外观复制

团队成员熟悉的是：

- 左侧对象树
- 中央编辑工作台
- 右侧检查与上下文信息
- 底部日志、结果、分析入口

AudioForge 应该复刻这套工作流骨架，而不是复制 Wwise 的颜色、图标或具体控件外观。

### 2. 任务聚焦优先于控件堆叠

中央区域必须围绕当前任务组织：

- 事件设计
- 资源整理
- 总线与混音
- 校验修复
- 构建交付

上下文信息改为收口到各工作区自身的上下文栏，不再固定占用右侧全局检视器，以避免窄窗口下的信息裁切和重复摘要。

### 3. 结果回看独立成坞

校验、构建、响度和日志需要始终有回看入口，但不应长期抢占主编辑空间。底部结果坞负责快速回看，完整分析仍保留结果中心。

## 当前推荐主布局

### 目标布局

- 左侧：工程浏览器
- 中央：当前工作区主编辑面
- 底部：结果坞
- 工作区内：按需显示的上下文栏

### 各区职责

左侧工程浏览器：

- 负责对象查找、层级组织、快速筛选和多选入口。

中央工作台：

- 只承担当前任务的主要编辑动作。
- 事件模式下聚焦参数和试听。
- 资源模式下聚焦片段和批处理。
- 总线模式下聚焦路由和输出。

工作区上下文栏：

- 在事件、资源、总线、校验和构建页内按需显示任务导航。
- 需要时显示工程摘要、当前对象头和对象引用关系。
- 与主编辑区之间保持可手动拖拽缩放，不固定占用全局右侧栏。

底部结果坞：

- 常驻显示当前模式、当前结果焦点和脏状态。
- 一键跳到日志、校验、构建、响度结果。
- 不承担复杂编辑，只承担回看与定位。

## 本轮原型已落点

- 顶部全局搜索文案扩大到对象、总线、问题和结果。
- 顶部全局搜索执行时已支持跨对象跳转，可直接打开工程对象、总线和结果条目。
- 顶部命令按钮已收敛为导航与隐藏动作面板，负责工作区切换、结果跳转和少量未显式暴露的动作入口，不再重复顶栏与工作区里已可见的执行按钮。
- 取消固定右侧检视器，将工程摘要、当前对象和对象引用回收到各工作区上下文栏。
- 事件工作区不再重复展示对象头和引用卡片。
- 总线工作区不再重复展示工程摘要卡片。
- 总线工作区已增加路由图，可同时显示当前总线、上游链路、下游分支和作者态输出。
- 资源工作区已增加最近批量反馈卡，批量权重、属性、重命名、排序和拖拽重排结果都会保留在当前事件上下文里。
- 底部结果区域从隐藏入口改为常驻结果坞。
- 底部结果坞已增加日志、校验、构建、响度四块结构化摘要，不再只有入口按钮。
