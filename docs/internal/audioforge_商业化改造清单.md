# AudioForge 商业化改造清单

> 当前状态：2026-04-30 已完成第一轮主界面产品化重构，AppShell、顶部应用栏、左侧任务导航、欢迎页、结果中心、六大工作区双栏结构、空状态卡片和结果状态色均已落地。本文档以下内容默认视为“剩余差距与后续项”，不再代表当前界面仍停留在最初状态。

## 文档目的

这份清单面向“把当前 AudioForge 从内部可交付工具，提升到商业软件级体验”的目标。

当前判断：

- 功能骨架：已经具备较强基础。
- 工程稳定性：已达到可交付基线。
- 主要差距：不在导出契约，而在设计系统、信息架构、专业控件、可视化表达、反馈系统和首次使用体验。
- 2026-05-09 决策补充：第二轮本地工具优化采用“Wwise 兼容工作台”策略，范围限定在桌面工具层，不改 Unity 导出契约。

这份清单按 P0 / P1 / P2 分层：

- P0：不做就很难达到商业软件门槛。
- P1：做完后会明显进入成熟产品阶段。
- P2：用于形成品牌感、专业感和市场竞争力。

## P0

### 1. 重构信息架构

目标：让用户按任务而不是按控件理解产品。

必须完成：

- 短期主工作台收敛为“左树 + 中工作台 + 工作区上下文栏 + 底部结果坞”，并保证这些分栏都可手动拖拽缩放，让熟悉 Wwise 的用户第一次进入就能建立位置感。
- 将当前“工程树 + 属性页 + 内容页 + 响度页 + 报告页”的控件式组织，重构为任务式组织。
- 明确一级工作模式：导入整理、事件编辑、音频与总线、校验修复、导出交付。
- 把“日志”“构建报告”“响度扫描”从长期驻留的大面板，改成任务结果中心或可折叠侧栏。
- 明确对象头部、属性编辑区、片段区、报告区之间的主次关系，去掉重复摘要。

验收标准：

- 新用户第一次打开后，能在 10 秒内理解当前页面正在做什么。
- 高频任务路径不需要跨 3 个以上页签。
- 熟悉 Wwise 的用户应能在 1 分钟内找到对象树、主编辑区、上下文检视区和结果回看区。

### 2. 建立设计系统

目标：从“单文件样式表”升级为“可维护的产品设计语言”。

必须完成：

- 定义颜色 token、间距 token、字号层级、圆角层级、边框层级、状态色。
- 定义按钮、输入框、标签页、面板头、芯片、提示条、空状态卡片的统一规范。
- 去掉以 `wwise_style` 为中心的视觉命名方式，改成 AudioForge 自己的设计语言命名。
- 为不同状态建立清晰语义：正常、聚焦、悬停、选中、禁用、错误、警告、成功。

验收标准：

- 所有页面共享同一套视觉 token。
- 新增页面不需要再复制大段样式字符串。

### 3. 升级关键编辑控件

目标：让核心编辑体验达到专业工具标准。

必须完成：

- 事件树从当前 QTreeWidget 能力，升级为更稳定的 model/view 方向。
- 片段表从当前 QTableWidget 填格方式，升级为可排序、可筛选、可局部刷新的 model/view 结构。
- 统一支持列配置、排序状态显示、批量编辑反馈、拖拽反馈、空列表提示。
- 对大工程场景建立性能基线，例如 1000+ Event、5000+ Clip 时仍保持顺滑。

验收标准：

- 大批量对象编辑无明显卡顿。
- 排序、筛选、批量编辑、拖拽重排彼此不会互相破坏状态。

### 4. 重做反馈系统

目标：从“日志/文本框驱动”升级为“产品化反馈”。

必须完成：

- 关键操作提供明确的即时反馈，如保存成功、校验完成、构建失败、导出完成。
- 校验问题支持按对象聚合、按严重程度筛选、按修复优先级排序。
- 构建差异从纯文本输出，升级为结构化摘要加可点击详情。
- 长任务统一走后台任务条，而不是让用户盯着日志区。

验收标准：

- 用户不打开日志，也能理解刚发生了什么。
- 出错时能快速定位对象、原因和下一步建议。

### 5. 重做空状态与首次使用体验

目标：让产品具备商业软件的“第一分钟体验”。

必须完成：

- 增加欢迎页：最近工程、示例工程、快速开始、文档入口。
- 未选中对象时显示引导卡片，而不是只有禁用态控件。
- 无片段、无总线、无导出目录、无构建结果等状态都要有专门空态。
- 首次启动提供简短引导，说明基本工作流。

验收标准：

- 用户第一次打开工具，不需要文档也能完成最小闭环。

## P1

### 6. 强化可视化表达

目标：把“音频工具感”真正做出来。

必须完成：

- 响度监视器增加刻度、参考线、峰值标识、双序列对比、悬停读数。
- 总线工作区增加路由图和有效输出链可视化，而不只是表单和标签。
- Event 区增加模式摘要、权重分布、实例上限与冷却的可视化摘要。
- 生成预览增加对象级 diff 视图，而不是只有文本。

验收标准：

- 用户查看总线和响度时，不需要读大量文字就能判断问题。

### 7. 建立命令面板与全局搜索

目标：提升高频操作效率。

必须完成：

- 增加命令面板，统一承载创建、跳转、校验、构建、导出、视图切换。
- 增加全局搜索，支持 Event、Bus、Clip、问题、报告结果的跨区跳转。
- 统一快捷键提示与帮助入口。

验收标准：

- 高频用户可以在不依赖鼠标的前提下完成主要操作。

### 8. 引入模板与预设工作流

目标：降低上手门槛，并提高团队一致性。

必须完成：

- 提供默认工程模板。
- 提供 Event 模板，例如 UI 点击、随机 SFX、BGM 切换、Combo 事件。
- 批量导入时支持模板规则和命名建议。

验收标准：

- 新项目启动时不需要从零搭结构。

### 9. 完善布局与工作区定制

目标：支持不同角色的工作方式。

必须完成：

- 支持多个预设工作区，如设计模式、批处理模式、交付模式。
- 支持保存与恢复自定义布局。
- 支持双屏或宽屏下更稳定的面板行为。

验收标准：

- 用户可以根据任务切换工作模式，而不是手动拖面板。

## P2

### 10. 打造品牌感与高级视觉方向

目标：形成“这是 AudioForge”的独立产品识别。

必须完成：

- 明确品牌关键词，例如精确、轻量、专业、现代。
- 重做图标风格与首页欢迎区。
- 建立统一插画、空状态图形、提示卡片视觉语言。
- 形成官网/文档/应用内部一致的视觉口径。

### 11. 增加专业分析工具

目标：提高高级用户粘性。

可选方向：

- 权重分布预览
- 事件命中模拟器
- 构建前差异比较器
- 批量规范检查器
- 项目健康度仪表盘

### 12. 做成完整产品闭环

目标：让软件具备持续商用能力。

必须补齐：

- 应用内更新提示
- 崩溃恢复与诊断包导出
- 偏好设置中心
- 用户帮助中心与示例资源
- 更正式的版本发布说明和迁移说明

## 建议排期

### 阶段 A

- P0.1 信息架构
- P0.2 设计系统
- P0.4 反馈系统
- P0.5 空状态与首次使用体验

### 阶段 B

- P0.3 核心控件升级
- P1.6 可视化增强
- P1.7 命令面板与全局搜索

### 阶段 C

- P1.8 模板工作流
- P1.9 工作区定制
- P2 全部

## 最终判断

当前版本已经有“内部专业工具”的底子，但还没有“商业软件”的产品完成度。

真正的分水岭不是继续加功能，而是：

- 把 UI 从控件堆叠改成任务流程
- 把视觉从手工样式改成设计系统
- 把反馈从日志文本改成结构化产品反馈
- 把可视化从辅助标签改成核心理解入口
