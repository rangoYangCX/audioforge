# AudioForge 0.09.0 - Schema3 AudioObject Release and Bus Workspace Cleanup

AudioForge 0.09.0 把当前分支的主线从“事件内嵌 Audio 子对象”正式推进到“项目级 AudioObject + Event 引用 AudioId”的交付口径：SchemaVersion 3、AudioObjects、Unity runtime 解析链、全链路校验和 SDK 文档已经全部切到同一套模型；同时 Bus 混音台工作区也完成了更适合日常编辑的分页整理。

## Highlights

- `AudioData.json` 已升级到 `SchemaVersion = 3`，顶层正式输出 `AudioObjects` 与 `Events[AudioId]`，不再保留嵌套 `Audio` 或扁平镜像字段。
- Event 现只保留动作层字段和 `AudioId`，Bus、PlayMode、Clips、GameSync 绑定及声音属性全部归 AudioObject 负责。
- Unity runtime、JSON 适配器与全链路检查已切到 Schema 3 单一路径：缺失 `AudioId`、缺失目标 `AudioObject` 或契约字段不完整会直接判失败。
- 编辑器对象浏览与导入流程已跟随主模型重排：Audio 树成为一等对象面，源音频树统计“引用 Audio 数”，外部拖拽导入按事件树 / Audio 树 / 源音频树三条规则收口。
- Bus 混音台工作区已拆为“当前 Bus / 工程总览 / Bus GameSync”三页；当前 Bus 再拆成“路由 / 电平与导出”二级页签，降低拥挤和误点成本。

## Compatibility

- 0.09.0 的对接文档与 SDK 包默认以 Schema 3 为准；若项目仍在使用 Schema 1 或 Schema 2，请继续沿用历史包，不要把旧运行时和新导出物混用。
- 若直接使用仓库内自带 Unity runtime，重点是同步最新包和文档；若维护自研 runtime，则必须补齐 `AudioObjects + Events[AudioId]`、GameSync 和 emitter context 的解析链。
