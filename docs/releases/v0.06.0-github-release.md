# AudioForge 0.06.0 - Diagnostic Center and Workspace Layout Stabilization

AudioForge 0.06.0 聚焦两件事：把桌面工具已有的日志、校验、构建、响度和 Bus 状态统一收进一个诊断入口，以及把近期工作台收口带来的布局回归彻底锁住。整个版本仍然遵守现有 Unity 接入主契约，不要求 Unity 项目改 SDK 接口或导出消费方式。

## Highlights

- 结果中心新增诊断概览页，直接汇总最近日志、校验、构建、响度和 Bus 上下文。
- `MainController` 新增结构化诊断快照模型，界面不再分别维护多份重复摘要。
- 构建交付页新增构建画像列表与详情，最近一次构建范围、资源差异和交付目标会直接展开。
- Bus 路由图与资源页批量反馈继续并入现有工作流，不新增平行模块。
- 修复工作区在事件设计、构建交付、校验修复和结果中心之间切换时中央编辑区被挤窄的问题，锁住内页签布局。

## Unity Compatibility

- Unity 现有接入主契约不变，仍以 `AudioData.json`、运行时音频资源和 `AudioEventID.cs` 为核心交付物。
- 0.06.0 的主要变化都发生在桌面工具侧的诊断聚合、工作台信息组织和布局稳定性上。
- 如果 Unity 同学已经在对接旧包，可以继续使用现有 SDK 包；只有需要同步新版文档、副本验证材料或重新导出的样例时，才需要重新生成 `AudioForgeUnityPackage-0.06.0`。

## Known Boundaries

- 这版仍然不改 Unity SDK 接口，也不改导出字段语义。
- 当前补的是编辑期诊断和桌面工作台稳定性，不是运行时 profiler、RTPC 或更重的中间件语义扩展。
