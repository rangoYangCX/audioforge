# AudioForge 0.09.1 - UPM SDK Output Spec and Editor Workspace Polish

AudioForge 0.09.1 把上一版已经落地的 Schema 3 / Audio Object 主模型，继续收口成一套更可交付的发布口径：Unity SDK 现在有了明确的 UPM 输出规范，Windows 桌面发布目录会直接内嵌 `com.audioforge.runtime` 本地包；同时编辑器侧补齐了片段编辑台布局和源音频树选择语义两处高频使用体验问题。

## Highlights

- 新增 `docs/unity/UnitySDK输出规范.md`，正式定义未来 Unity SDK 必须按 `com.audioforge.runtime` UPM 根目录结构输出：`package.json`、`README.md`、`Runtime/`、`Editor/`、`Documentation~/`。
- `tools/package_unity_upm_sdk.py` 现会把 `UnitySDK输出规范.md` 同步进包内 `Documentation~/Docs/Canonical/`，保证仓库文档、包内文档和交付目录是同一套口径。
- `tools/build_windows_exe.py` 现在会把当前版本的 UPM SDK 一并嵌入 `dist/AudioForge-0.09.1-windows/SDK/com.audioforge.runtime/`，减少 Unity 程序从多个目录手动拼包的成本。
- 片段编辑台布局已收口为波形优先，下方操作区与批量反馈卡片改为更紧凑的工作台结构，避免波形区被说明和按钮长期挤压。
- 源音频树默认选择模式改为 `ExtendedSelection`，单击默认单选，`Ctrl` / `Shift` 仍可多选，行为更接近资源管理器。

## Compatibility

- 0.09.1 不改变当前 `SchemaVersion = 3`、`AudioObjects + Events[AudioId]`、GameSync 和 emitter context 的运行时契约；它主要补的是 SDK 交付规范、包内文档一致性和编辑器可用性。
- 若项目仍在使用 0.09.0 之前的历史包，可继续沿用旧运行时；若切到 0.09.1，建议同时同步最新对接文档、输出规范和包内 Quick Start。
