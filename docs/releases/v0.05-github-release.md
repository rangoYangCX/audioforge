# AudioForge 0.05 - Incremental and Selected Build Workflow

AudioForge 0.05 聚焦大工程下的构建效率和可追踪性。桌面工具现已支持全量构建、增量构建和选中构建三种范围，并在不改变 Unity 现有接入主契约的前提下，引入资源级复用、构建计划预览和版本化变更记录。

## Highlights

- 新增全量构建、增量构建、选中构建三种范围，减少大工程下不必要的全量重建。
- 构建交付页新增构建计划预览，可直接看到请求范围、实际执行范围、重建资源数、复用资源数和移除资源数。
- 导出器现会全量刷新 `AudioData.json`、`AudioManifest.json`、`AudioEventID.cs`，同时复用未变化音频资源，只重建脏资源。
- `AudioManifest.json` 新增 `BuildFingerprint`，`BuildReport.json` 新增 `BuildPlan`，方便联调、审计和 CI 跟踪构建差异。
- 选中构建若发现选区外脏资源，会自动升级为增量构建，优先保证完整包一致性。

## Unity Compatibility

- Unity 现有接入主契约不变，仍以 `AudioData.json`、运行时音频资源和 `AudioEventID.cs` 为核心交付物。
- `BuildFingerprint` 和 `BuildPlan` 属于新增辅助信息，旧接入方即使暂时不消费这些字段，也不会影响既有播放链路。
- 这次升级主要改善工具侧构建效率和可追踪性，不要求 Unity 项目立刻修改运行时代码。

## Known Boundaries

- Unity 侧当前仍定位为参考运行时，正式项目接入前建议至少完成一轮目标工程联调。
- 选中构建的一期目标是减少无关资源重建，而不是导出一个可独立运行的局部子包。
