# AudioForge 0.05

这是一版围绕大工程构建效率收口的发布。0.05 将桌面工具的导出流程升级为“全量构建 / 增量构建 / 选中构建”三种范围，并在不破坏 Unity 现有接入契约的前提下，把资源级复用、构建计划可视化和版本文档一起补齐。

## 本版重点

- 新增全量构建、增量构建、选中构建三种范围，适合大工程下只处理本次改动所需内容。
- 构建交付页新增构建计划预览，可直接看到请求范围、实际执行范围、重建资源数、复用资源数、移除资源数。
- RuntimeExporter 现在会全量刷新 `AudioData.json`、`AudioManifest.json`、`AudioEventID.cs`，同时对未变化音频资源做增量复用，只重建脏资源。
- `AudioManifest.json` 新增 `BuildFingerprint`，`BuildReport.json` 新增 `BuildPlan`，便于联调、审计和 CI 追踪构建差异。
- 选中构建不会产出残缺子包；如果检测到选区外仍有脏资源，会自动升级为增量构建，优先保证完整包一致性。

## 对 Unity 接入方的影响

- 现有 Unity 接入主契约不变，仍以 `AudioData.json`、运行时音频资源和 `AudioEventID.cs` 为核心交付物。
- `BuildFingerprint` 和 `BuildPlan` 属于新增辅助信息，旧接入方即使暂时不消费这些字段，也不会影响既有播放链路。
- 这次升级主要改善工具侧构建效率和可追踪性，不要求 Unity 项目立刻修改运行时代码。
