# AudioForge 0.07.0 - Phase2 Editor Delivery and SDK Delta Clarification

AudioForge 0.07.0 把这轮 phase2 编辑器交付正式收口成可交接版本：对象浏览器升级为三分页，事件树 bindings 编辑入口收口为弹窗，`PlayMode` 明确补齐 `OneShot`，同时把 Unity 对接文档和 SDK 包内文档改成更适合“从第一期直接接到当前版本”的阅读方式。

## Highlights

- 左侧对象浏览器升级为“总线树 / 源音频树 / 事件树”三分页，源音频浏览、引用状态与事件定位统一收口。
- 事件树 bindings 编辑入口统一为弹窗，支持追加、替换、拖拽追加反馈，以及 `Enabled` / `Active` 状态切换。
- `PlayMode` 正式补齐 `OneShot`，单源音频自动建事件时默认落为 `OneShot`；试听、校验和导出都按有效 Clip 集合工作。
- 工程设置新增“根据事件命名智能分配总线”开关，并纳入 `.afproj` 保存。
- 仓库主文档和 SDK 包内文档都新增“一期到当前变化总览”，Unity 同学不必再从实现细节里反推差异。

## Compatibility

- 0.07.0 不要求 Unity 侧去读取 editor-only 字段，也不要求新增另一套运行时目录结构。
- 参考运行时代码仍可继续消费新版导出结果；真正需要同步的是对 `PlayMode = OneShot`、有效 Clip 集合和 editor-only 边界的理解。
- 如果 Unity 同学已经拿到旧包，建议至少同步新版 SDK 文档和重新打包后的交付目录，以便获得最新 canonical 文档、副本验证材料和签收摘要。

## Validation

- `pytest`：112 项通过。
- OneShot 预览、导出和校验相关回归测试已补齐。
- Unity 包重新打包后会附带最新全链路检查报告和 release sign-off。
