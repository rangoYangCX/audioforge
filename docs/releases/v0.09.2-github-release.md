# AudioForge 0.09.2 - Preview Center and Schema 3 Release Baseline

AudioForge 0.09.2 把当前编辑器和 Unity runtime 的主交付口径继续往“可冻结、可签收、可交付”推进了一步。桌面工具侧，底部区域已经收口为完整试听中心，减少与当前工作流弱相关的编辑入口干扰；运行时侧，Unity API 补齐了更直接的 emitter GameSync 调用；交付侧，Schema 3、Audio Object 主模型与当前 SDK 文档命名已经统一到同一套版本锚点。

## Highlights

- 底部区域已重构为完整试听中心，移除了波形编辑、Bus 视图和结果卡片等非核心入口，让试听、切换和观察反馈聚焦在同一个工作面。
- 试听 GameSync 条现在会直接显示当前命中的作用域来源，减少把全局、对象级和 emitter 级 GameSync 混读的误判。
- Unity runtime 新增 `SetEmitterGameParameter` 等 emitter context API，旧接口保留兼容转发，便于项目逐步迁移。
- Unity SDK 文档已同步到新的命名与交付口径，仓库文档、包内文档和当前版本锚点保持一致。
- 片段编辑台补齐 `wide / medium / compact` 响应式布局，修复重排过程中误销毁 Qt 控件的问题。

## Compatibility

- 0.09.2 不改变当前 `SchemaVersion = 3`、`AudioObjects + Events[AudioId]`、GameSync 主契约与导出结构。
- 现有 0.09.x Unity runtime 接入项目可继续沿用当前导出协议；如果同步到 0.09.2，建议同时更新 Unity 对接文档、输出规范和包内说明。
- 当前版本的重点是收口预览与交付基线，不是引入新的导出格式分叉。
