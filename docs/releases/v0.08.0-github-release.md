# AudioForge 0.08.0 - Phase3 GameSync Release and Event/Audio Contract Lock-in

AudioForge 0.08.0 把这轮 phase3 主链和事件/Audio 职责拆层正式收口为可交付版本：GameSync 已贯穿 authoring、preview、exporter 和 Unity runtime，事件对象回到“播放动作层”，声音逻辑正式收拢到内嵌 `Audio` 子对象。

## Highlights

- 事件模型正式拆成“事件动作层 + Audio 子对象层”：`CooldownSeconds`、`MaxInstances`、`StealPolicy` 保留在事件顶层，`Bus`、`PlayMode`、`AvoidImmediateRepeat`、音量/音高、Combo、Clips 和 GameSync 绑定统一进入 `Audio`。
- `AudioData.json` 的 `SchemaVersion = 2` 事件 payload 现正式带嵌套 `Audio` 对象，同时保留扁平镜像字段作为兼容桥和审计锚点。
- 事件设计页已按相同边界重排为“事件元数据 / 播放控制 / Audio 属性 / Audio 调制”，不再把 Audio 模式和选择逻辑误放在事件动作层里展示。
- Unity runtime 与 JSON 适配器已优先消费事件内嵌 `Audio`，但仍保留旧扁平字段回退路径，方便现有项目平滑过渡。
- 全链路检查现把嵌套 `Audio` 与扁平镜像一致性纳入发布态硬校验，导出契约一旦漂移会在发布前直接失败。

## Compatibility

- 0.08.0 仍保留旧 `.afproj` 的扁平事件字段读取兼容，现有工程无需手工迁移即可打开。
- Unity 侧若直接使用仓库自带 runtime，一般不需要新增第二套播放代码；重点是按新文档理解“事件顶层 = 播放控制，`Audio` = 声音对象”。
- 扁平事件字段仍会继续导出一段时间，方便旧工具链、审计脚本和过渡期运行时继续工作，但新代码应优先读取 `Audio`。
